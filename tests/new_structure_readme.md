# My-CV-Engineering-Projects

---

# 1. Описание

--- 
- **Название проекта**: My-CV-Engineering-Projects
- **Кодовое имя**: My-CV-Engineering-Projects
- **Тип проекта**: Open-source портфолио для CV-разработчика
- **Версия**: MVP
- **Лицензия**: MIT
- **Целевая аудитория**: рекрутеры, computer vision engineer, начинающие разработчики
- **Платформа**: GitHub
- **Основной язык интерфейса**: русский

Проект `My-CV-Engineering-Projects` представляет собой портфолио для самопрезентации начинающего инженера по компьютерному(техническому) зрению (CV engineer). Включает серию учебных проектов по обработке изображений, распознаванию объектов и компьютерному зрению на Python.

# 2. Цель

**Цель проекта** — создать публичное портфолио для демонстрации навыков, реализовать серию проектов по компьютерному зрению, упростить и ускорить разработку с помощью ИИ-ассистента и настроить эффективный workflow разработки.

# 3. Назначение и роль папок и файлов в проекте

---

## 3А. Корневые папки и файлы

- **`.codeassistant/`**, **`.continue/`**, **`.gigaide/`**, **`.idea/`**, **`.vscode/`** — служебные папки и файлы, используемые в различных средах разработки.

- **`ai-conversations/`** — хранит все записи разговоров с AI-ассистентом в формате Markdown, что позволяет отслеживать ход мыслей и принятые решения.

- **`anaconda_projects/`** — содержит проекты, созданные в среде Anaconda, популярной платформе для научных вычислений и машинного обучения.

- **`assets/`** — папка для статических ресурсов, таких как изображения, иконки, шрифты и другие медиафайлы, используемые в проектах.

- **`build/`** — временные файлы сборки приложений, генерируемые при компиляции или упаковке проектов.

- **`CV-000-ai-assistant/`** — подпроект;

- **`CV-010_face_detection/`** — подпроект;

- **`CV-011-QR-code-recognition/`** — подпроект;

- **`CV-012-image-filters/`** — подпроект;

- **`CV-013-CNN-classification/`** — подпроект;

- **`CV-014-transfer-learning/`** — подпроект;

- **`CV-015-YOLO-detection/`** — подпроект;

- **`CV-016-U-Net-segmentation/`** — подпроект;

- **`CV-017-style-transfer/`** — подпроект;

- **`dist/`** — финальные скомпилированные бинарные файлы приложений (например, .exe), готовые к распространению.

- **`docs/`** — центральная папка документации проекта, включающая руководства, правила работы, описание проекта и другие важные документы.

- **`scripts/`** — скрипты автоматизации на PowerShell и batch, предназначенные для управления окружением, запуска сервисов, синхронизации и тестирования.

- **`static/`** — статические файлы веб-приложений, такие как CSS, JavaScript и результаты обработки изображений.

- **`teamplate project/`** — шаблон для создания новых CV-проектов с готовой структурой папок, документацией и конфигурацией.

- **`tests/`** — папка для тестов, временных файлов и чек-листов, используемых для проверки и завершения проектов.

- **`LICENSE`** — файл лицензии MIT, определяющий условия использования и распространения проекта.

- **`README.md`** — главный файл документации проекта, содержащий общую информацию, описание, структуру и инструкции по установке.

- **`.gitattributes`**, **`.gitignore`**, **`.gitignore.txt`** — служебные файлы Git для управления версиями, настройки атрибутов и исключения файлов из репозитория.
