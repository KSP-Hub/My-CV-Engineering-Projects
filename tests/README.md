# My-CV-Engineering-Projects

## Общая информация
- **Название проекта**: My-CV-Engineering-Projects
- **Тип проекта**: Open-source портфолио для CV-разработчика
- **Версия**: MVP
- **Лицензия**: MIT
- **Целевая аудитория**: рекрутеры, computer vision engineer, начинающие разработчики
- **Платформа**: GitHub
- **Основной язык интерфейса**: русский

## Описание проекта
Проект представляет собой портфолио для самопрезентации начинающего инженера по техническому зрению. Включает серию учебных проектов по обработке изображений, распознаванию объектов и компьютерному зрению на Python.

## Скрипты в папке scripts/

### Основные скрипты управления AI-ассистентом:

- **`start_cv_assistant.ps1`** — запускает локальный сервер Ollama с оптимизированной моделью для CV-проектов на GPU Quadro RTX 4000
- **`check_assistant.ps1`** — проверяет состояние сервера Ollama, порта и доступности модели
- **`reset_assistant.ps1`** — аварийный перезапуск сервера Ollama и освобождение порта 11434
- **`sync-gitignore.ps1`** — синхронизирует .gitignore с .gitignore.txt для обхода бага в AI-ассистенте
- **`create-ai-note.ps1`** — создаёт markdown-файл для записи разговора с AI в папке ai-conversations/

### Скрипты автоматизации и синхронизации:

- **`sync_all.ps1`** — основной скрипт синхронизации всех скриптов в нужные места
- **`sync_to_cloud.ps1`** — создаёт архив контекста проекта для загрузки в облако
- **`sync_from_cloud.ps1`** — загружает и распаковывает контекст проекта из облака
- **`cloud_context_manager.ps1`** — менеджер контекста для работы с облаком (статус, синхронизация, загрузка)
- **`save_project_context.ps1`** — сохраняет текущий контекст проекта в документ
- **`check_context.ps1`** — проверяет наличие файлов контекста и статус проекта

### Вспомогательные скрипты:

- **`check_continue_config.ps1`** — проверяет конфигурацию Continue в VS Code
- **`Modelfile-rtx4000.ps1`** — создает оптимизированную модель Ollama для Quadro RTX 4000
- **`start_cv.bat`** — запускает проект в VS Code
- **`launch_notebook.bat`** — запускает Jupyter Notebook для анализа данных

## Описание папок проекта

### Корневые папки:

- **`.codeassistant/`, `.continue/`, `.gigaide/`** — конфигурационные файлы различных AI-инструментов
- **`ai-conversations/`** — хранит все записи разговоров с AI-ассистентом в формате .md
- **`docs/`** — центральная документация проекта, включая project-docs/ с ключевыми документами
- **`scripts/`** — все скрипты автоматизации, разделённые по функциональности
- **`tests/`** — тесты и временные файлы, включая чек-листы для завершения проектов

### Подпроекты:

- **`CV-000-ai-assistant/`** — завершённый проект по настройке AI-помощника
- **`CV-010_face_detection/`** — проект по детекции лиц с OpenCV, включает веб-интерфейс на Flask
- **`teamplate project/`** — шаблон для создания новых CV-проектов с готовой структурой и документацией

### Технические папки:

- **`build/`, `dist/`** — временные и финальные файлы сборки приложений
- **`assets/`, `static/`** — статические ресурсы, изображения, CSS/JS файлы
- **`anaconda_projects/`** — проекты, созданные в Anaconda

## Технологии и инструменты
- **Python** — основной язык разработки
- **OpenCV** — обработка изображений и компьютерное зрение
- **Flask** — веб-фреймворк для создания интерфейсов
- **Ollama** — запуск локальных LLM моделей
- **Git / GitHub** — управление версиями
- **Git LFS** — работа с большими файлами
- **Jupyter Notebook** — анализ данных и прототипирование
- **PowerShell** — автоматизация на Windows

## Установка
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/KSP-Hub/My-CV-Engineering-Projects.git
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Установите Git LFS:
   ```bash
   git lfs install
   ```
