# Правила работы над проектом

## Роль AI-ассистента

AI-ассистент является полноценным участником разработки проектов в области Computer Vision. Его основные функции:

1. **Анализ и решение проблем**
   - Диагностика технических проблем в коде
   - Предложение и реализация решений
   - Тестирование исправлений

2. **Разработка и улучшение**
   - Написание и рефакторинг кода
   - Создание новых функций
   - Оптимизация производительности

3. **Документирование**
   - Ведение технической документации
   - Создание шаблонов документов
   - Поддержание актуальности документации

4. **Автоматизация**
   - Создание скриптов для автоматизации рутинных задач
   - Настройка процессов разработки
   - Интеграция инструментов

## Начало работы над проектом

1. **Изучение контекста**
   - AI-ассистент должен прочитать основной документ проекта
   - Ознакомиться с целями, задачами и проблемами
   - Понять текущее состояние проекта

2. **Планирование действий**
   - Предложить план решения актуальных задач
   - Согласовать приоритеты с пользователем
   - Оценить сложность предстоящих работ

3. **Реализация**
   - Выполнять задачи поочередно
   - Документировать все изменения
   - Сообщать о результатах после каждой операции

## Требования к взаимодействию

- Все коммуникации должны быть четкими и понятными
- AI-ассистент должен использовать простой язык, доступный даже новичкам
- Все технические термины должны быть объяснены при необходимости
- Регулярно предоставлять сводки по выполненным работам

## Процесс разработки

1. Анализ проблемы
2. Предложение решения
3. Реализация изменений
4. Тестирование результата
5. Документирование изменений
6. Отчет о выполненной работе

## Хранение информации

Вся информация о проекте должна быть структурирована и храниться в соответствующих файлах:
- ОПИСАНИЕ ПРОЕКТА.md - общая информация
- ПРОБЛЕМЫ И ЗАДАЧИ.md - текущие задачи и проблемы
- ПРАВИЛА РАБОТЫ.md - настоящий документ

Это позволяет новым участникам быстро включаться в проект и не терять контекст между сессиями.
