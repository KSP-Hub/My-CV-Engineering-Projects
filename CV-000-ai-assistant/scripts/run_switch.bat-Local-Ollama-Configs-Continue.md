# О скрипте "run_switch.bat"
Файл run_switch.bat является частью системы управления моделями для AI-ассистента Continue, которая позволяет переключаться между локальной и облачной версиями языковых моделей.
Основная функция run_switch.bat - запуск скрипта smart_switch.py, который предоставляет пользователю интерактивное меню для выбора типа модели. Вот что делает этот инструмент:


1. **Предоставляет выбор между двумя режимами работы:**

- Локальная модель: использует deepseek-coder:6.7b-instruct-q4_K_M через Ollama, работает автономно без интернета, оптимизирована для CV-задач
- Облачная модель: использует deepseek-chat через API DeepSeek, требует интернет-подключения, но может быть более "умной"

2. **Автоматически обновляет конфигурацию:**

- При выборе опции скрипт создает или обновляет файл конфигурации ~/.continue/config.json
- В зависимости от выбора подставляет соответствующие настройки модели

3. **Имеет режим автопереключения**:

- Проверяет лимиты использования облачной модели
- При исчерпании лимитов может автоматически переключать на локальную модель

4. **Ведет логирование:**

- Записывает все переключения в файл switch_log.txt с отметкой времени

5. **Оптимизирован для конкретного оборудования**:

- Учитывает конфигурацию Dell Precision 7920 с 2x Xeon, 128GB RAM и Quadro RTX 4000
- Специализирован на задачи компьютерного зрения (OpenCV, PyTorch)

- После переключения требуется перезапустить VS Code для применения изменений. Этот инструмент позволяет гибко управлять ресурсами и возможностями AI-ассистента, выбирая между скоростью локальной обработки и "умом" облачной модели в зависимости от текущих задач и ограничений.

# Содержание архива

Скачайте архив run_switch.bat-run_switch.bat-Local-Ollama-Configs-Continue.rar и сохраните его в папку с config. Содержание архива:
1. switch_log.txt
2. config.json
3. run_switch.bat
4. smart_switch.py
